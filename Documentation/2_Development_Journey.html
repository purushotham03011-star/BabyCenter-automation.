"Now that we know the mission, let's talk about the tools in our belt. Why did we choose Python? How does the robot 'think'?"

1. Choosing the Tech Stack

We chose the "Power Trio" for modern automation:

  • Python: The language of data. Simple, powerful, and has great libraries.
  • Streamlit: Allowed us to build a beautiful Web Dashboard in just 100 lines of code. No HTML/CSS needed!
  • Supabase: A database that acts like a spreadsheet but scales like Google. Perfect for storing our articles.

2. The Challenge: "The Maze"

Websites are like mazes. If the robot just turns left at every link, it might get lost in the "Terms of Service" or "Login" pages forever.

The Solution: Priority Queue

We gave our robot a "Brain". When it sees links, it sorts them:

If link says "See All" -> Go there FIRST (Priority!) If link says "Privacy Policy" -> Ignore it. If link looks like an Article -> Save it for later.

This ensures we dig DEEP into the content (First Trimester -> Week 12) instead of getting stuck on the surface.

3. The "Heuristic" Content Filter

How does a robot know what an article is?

We wrote a rule: "If a page has more than 500 characters of text in the main body, it's worth saving."

If it's too short, it's probably just a menu page. We still check it for links, but we don't clog up our database with empty pages.

4. Handling Updates (Upsert)

We used a smart database command called upsert (Update + Insert). It means:

"Hey Database, here is an article. If you've never seen this URL, add it. If you HAVE seen it, just update the text."
