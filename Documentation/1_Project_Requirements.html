1. The Problem

Imagine you want to build a knowledge base about "Pregnancy" for your own website. Using BabyCenter.com as a source is great, but manually copying 500+ articles would take weeks.

The Goal: Create a "Robot" (Automation) that does this for us automatically.

2. Specific Requirements

The client (you!) didn't just want a simple page reader. You needed something smarter:

  • The "Deep Dive": The robot shouldn't just read the front page. It must click "See All", "Trimesters", and "Weeks" to find the hidden articles deep inside.
  • Smart Extraction: It needs to tell the difference between a "Menu Page" (just links) and an "Article Page" (actual advice) and only save the articles.
  • Self-Healing Database: If an article changes on the website tomorrow, our database should update automatically without creating duplicates.

3. The Technical Wishlist

To make this usable for non-coders, we needed:

  • A Dashboard: A visual screen to enter URLs and watch the robot work live.
  • Supabase Integration: A modern cloud database to store the data securely.
  • Background Mode: A way to let it run while we sleep (Task Scheduler).
Mentor's Takeaway:
Great software starts with clear requirements. We aren't just "scraping a site"; we are building an Intelligent Content Aggregation Pipeline.
