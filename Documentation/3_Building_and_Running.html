"You have the code. Now, how do you make it come alive on a new machine? Here is your step-by-step field guide."

Step 1: The Foundation (Dependencies)

Just like a construction site needs raw materials, our Python program needs "Libraries". We list these in a file called requirements.txt.

To install them, we run the "Pip" command:

pip install -r requirements.txt

This downloads the ability to scrape (BeautifulSoup), talk to the web (Requests), and show the dashboard (Streamlit).

Step 2: The Keys (Configuration)

We never hard-code passwords! We use a special file called .env (Environment Variables).

Inside this file, you safely keep your:

  • SUPABASE_URL: Where your database lives.
  • SUPABASE_KEY: The password to open it.

Step 3: Launching the Dashboard

To start the visual control center, we simplified the command into a "One-Click" file: Run_App.bat.

Under the hood, it runs:

streamlit run scraper_app.py

This spins up a local web server on your computer that you can access via your browser.

Step 4: Going "Headless" (Automation)

We don't want to keep the browser open forever. So we built auto_updater.py.

This is a "Ghost" version of our app. It has no screen. It just wakes up, does the work, and closes. Use Windows Task Scheduler to run this file every morning.
